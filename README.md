# ⚡ ChatSQL — Conversational AI Assistant for Complex SQL

A full-stack AI-powered SQL assistant. Ask questions in plain English,
get SQL generated by LLaMA 3 via Groq, executed against your real database,
with results displayed inline — all in a beautiful conversational UI.

---

## 🚀 Quick Start

### 1. Clone / Place the project
```bash
cd chatsql
```

### 2. Create a virtual environment
```bash
python -m venv venv
source venv/bin/activate        # Linux/Mac
venv\Scripts\activate           # Windows
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

For your specific database driver:
```bash
pip install psycopg2-binary      # PostgreSQL
pip install pymysql              # MySQL
pip install pyodbc               # MSSQL
# SQLite needs no extra driver
```

### 4. Configure environment
```bash
cp .env.example .env
```

Edit `.env`:
```env
GROQ_API_KEY=your_key_from_console.groq.com
DATABASE_URL=postgresql://user:password@host:5432/dbname
GROQ_MODEL=llama3-70b-8192
SECRET_KEY=your-random-secret
```

Get your free Groq API key at: https://console.groq.com

### 5. Run
```bash
python app.py
```

Open http://localhost:5000

## 🔑 Environment Variables

| Variable | Required | Description |
|---|---|---|
| `GROQ_API_KEY` | ✅ | Your Groq API key |
| `DATABASE_URL` | ✅ | SQLAlchemy connection URL |
| `GROQ_MODEL` | Optional | Model name (default: llama3-70b-8192) |
| `SECRET_KEY` | Optional | Flask session secret |

|---|---|

| SQLite | `sqlite:///./mydb.sqlite` |
| MSSQL | `mssql+pyodbc://user:pass@host/db?driver=ODBC+Driver+17+for+SQL+Server` |

## ✨ Features

- **Conversational multi-turn SQL** — context preserved across questions
- **Auto SQL generation** — LLaMA 3 writes SQL from plain English
- **Live execution** — query runs against your real DB instantly
- **Inline results** — paginated table directly in the chat
- **Charts** — Bar, Line, Pie, Donut with one click
- **Export** — CSV, JSON, Excel
- **Edit & Re-run** — modify generated SQL inline
- **AI error fixing** — when a query fails, Groq suggests a fix
- **Schema browser** — live tree of all tables & columns, click to insert
- **Voice input** — speak your question (Chrome)
- **Conversation history** — persisted in localStorage
- **Settings** — switch models, rows per page, auto-chart

## 🔒 Security Notes

- The app uses **Read-Only** mode by default at the UI level, but the backend
  executes whatever SQL Groq generates. For production, consider:
  - Creating a read-only database user for the connection
  - Adding SQL query validation / allowlisting
  - Adding authentication to the Flask routes
- Never commit your `.env` file

## 🛠️ API Endpoints

| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/status` | Check DB + Groq connectivity |
| GET | `/api/schema` | Get full database schema as JSON |
| POST | `/api/ask` | Main: NL question → SQL → execute → results |
| POST | `/api/explain` | Explain a SQL query in plain English |
| POST | `/api/run_sql` | Execute raw SQL directly |
| POST | `/api/connect` | Connect to a new database at runtime |
